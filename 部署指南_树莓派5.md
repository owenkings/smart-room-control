## 1. 烧录系统

### 1.1 下载 Raspberry Pi Imager

在你的 **Windows 电脑**上下载并安装：  
https://www.raspberrypi.com/software/

### 1.2 烧录步骤

1. 将 TF 卡（建议 32GB Class 10）插入读卡器，连接电脑
2. 打开 Raspberry Pi Imager
3. 点击 **选择设备** → 选择 `Raspberry Pi 5`
4. 点击 **选择操作系统** → `Raspberry Pi OS (64-bit)`（带桌面版，方便调试）
5. 点击 **选择存储** → 选择你的 TF 卡
6. 点击右下角 **齿轮图标（高级设置）**，配置以下内容：

   | 选项 | 填写内容 |
   |------|---------|
   | 主机名 | `smartroom` |
   | 用户名 | `pi` |
   | 密码 | 自定义（记住它） |
   | 配置 Wi-Fi | 填写你的 Wi-Fi 名称和密码 |
   | 启用 SSH | ✅ 勾选（使用密码登录） |
   | 时区 | `Asia/Shanghai` |

7. 点击 **保存** → **烧录**，等待完成（约 5 分钟）

## 2. 首次开机配置

### 2.1 启动树莓派

1. 将烧录好的 TF 卡插入树莓派 5 的卡槽（底部）
2. 连接 USB-C 电源（5V 3A 以上）
3. 等待约 1 分钟，绿灯闪烁后系统启动完成

### 2.2 找到树莓派的 IP 地址

**方法一（推荐）**：在路由器管理页面查看，设备名为 `smartroom`

**方法二**：在树莓派接上显示器和键盘，打开终端输入：
```bash
hostname -I
```

### 2.3 SSH 连接（在 Windows 电脑上操作）

打开 Windows 的 **命令提示符** 或 **PowerShell**：
```bash
ssh admin@192.168.x.x
# 将 192.168.x.x 替换为实际 IP
# 输入密码（烧录时设置的密码）
```

> 后续所有命令均在 SSH 终端中执行，除非特别说明。

## 3. 安装系统依赖

### 3.1 更新系统

```bash
sudo apt update && sudo apt upgrade -y
```

> 首次更新可能需要 5~15 分钟，耐心等待。

### 3.2 启用必要接口

```bash
sudo raspi-config
```

进入菜单后依次操作：

| 路径 | 操作 |
|------|------|
| `Interface Options` → `I2C` | 选 `Yes`（BH1750 光照传感器用） |
| `Interface Options` → `SPI` | 选 `Yes`（备用） |
| `Interface Options` → `Camera` | 选 `Yes`（如使用 CSI 摄像头） |

操作完成后选 `Finish`，**重启**：
```bash
sudo reboot
```

重启后重新 SSH 连接。

### 3.3 安装系统级依赖

```bash
# OpenCV 依赖（Raspberry Pi OS Bookworm / Debian 12 适用）
sudo apt install -y \
  python3-pip python3-venv git \
  libopencv-dev python3-opencv \
  libopenblas-dev \
  libjpeg-dev libpng-dev libtiff-dev \
  libavcodec-dev libavformat-dev libswscale-dev \
  libv4l-dev v4l-utils \
  i2c-tools

sudo apt install -y libhdf5-dev


# 验证 I2C 是否启用
sudo i2cdetect -y 1
# 如果 BH1750 已接线，应该能看到地址 0x23
```

> **说明**：`libatlas-base-dev` 在 Bookworm 中已被 `libopenblas-dev` 替代，两者功能相同（BLAS 线性代数库），NumPy/OpenCV 均可正常使用。

### 6.0 面包板电源模块准备

**示意图：**

![面包板电源模块接线](assets/wiring_breadboard_power.png)

MB-102 面包板电源模块是整个硬件系统的供电核心，先把它配置好。

**步骤：**

1. 将 MB-102 电源模块插入面包板顶端（对齐电源轨）
2. 用 DC 电源适配器（6~12V）或 USB 线供电
3. 设置跳线帽：
   - **左侧轨道**跳线拨到 `3.3V` → 供传感器（DHT22、BH1750、光敏、红外接收）
   - **右侧轨道**跳线拨到 `5V` → 供舵机、继电器
4. 拨动电源开关到 ON，红色 LED 亮起表示正常

**面包板布局建议：**

```
┌─────────────────────────────────────────┐
│  [+3.3V─────────────────────────────+]  │  ← 左轨 3.3V（传感器）
│  [─GND───────────────────────────────]  │
│                                         │
│  区域A: DHT22 + BH1750    区域B: 红外   │
│                                         │
│  区域C: S8050三极管电路   区域D: 预留   │
│                                         │
│  [─GND───────────────────────────────]  │
│  [+5V────────────────────────────────+] │  ← 右轨 5V（舵机/继电器）
└─────────────────────────────────────────┘
         ↕ 杜邦线连接树莓派 GPIO
```

### 6.2 舵机 MG90S + 继电器模块

**示意图：**

![舵机与继电器接线](assets/wiring_servo_relay.png)

#### 🔵 舵机 MG90S（灯光开关）

| 舵机引线颜色 | 连接到 | 说明 |
|------------|--------|------|
| 橙色（信号） | GPIO18（Pin 12） | PWM 控制信号 |
| 红色（VCC） | 5V（Pin 2 或 4） | **必须接 5V，不能接 3.3V** |
| 棕色（GND） | GND（Pin 6） | 公共地 |

**舵机角度说明：**
```
中立位置（松开）: 90°
按下开关（开灯）: 60°   ← config.py: SERVO_ANGLE_ON = 60
按下开关（关灯）: 120°  ← config.py: SERVO_ANGLE_OFF = 120
```

> 实际角度需根据安装位置微调，先在 `config.py` 里改小范围（如 85°/95°）测试，再逐步加大。

**安装方式：** 用双面胶或扎带将舵机固定在墙壁开关旁，推杆对准开关按钮，确保中立位时推杆不接触开关。

#### 🟡 继电器模块（电源控制）

| 继电器引脚 | 连接到 | 说明 |
|-----------|--------|------|
| VCC | 5V（Pin 4） | 模块供电 |
| GND | GND（Pin 14） | 公共地 |
| IN | GPIO27（Pin 13） | 控制信号（低电平触发） |

**COM/NO/NC 端子说明：**
```
COM ── 公共端
NO  ── 常开端（继电器断开时断路，吸合时通路）← 接负载
NC  ── 常闭端（继电器断开时通路，吸合时断路）

演示接法: COM → 小风扇/LED灯带正极，NO → 5V，负极 → GND
```

> ⚠️ **继电器仅用于低压设备演示（5V/12V），严禁直接接 220V 市电。**

### 6.3 红外发射管 + 红外接收管 VS1838B

**示意图：**

![红外接线](assets/wiring_ir.png)

#### 🔴 红外发射管（空调控制）

通过 S8050 NPN 三极管驱动，发射距离可达 5 米：

```
接线方式（面包板搭建）:

GPIO25 (Pin22) ──[1kΩ电阻]──┬── 三极管 S8050 基极 (B)
                             │
                    三极管集电极 (C) ──[红外发射管负极(短脚)]
                    红外发射管正极(长脚) ── 5V (面包板右轨)
                    三极管发射极 (E) ── GND

识别三极管引脚（S8050，平面朝向自己，从左到右）:
  左脚 = E（发射极）
  中脚 = B（基极）
  右脚 = C（集电极）

识别红外发射管:
  长脚 = 正极（阳极）→ 接 5V
  短脚 = 负极（阴极）→ 接三极管集电极
```

> 没有三极管时可直接接：GPIO25 → [100Ω电阻] → 发射管正极，负极 → GND，但距离只有 1~2 米。

**安装位置：** 将发射管固定在摄像头盒子或项目板上，对准空调接收窗（通常在空调正面右下角）。

#### 🟢 红外接收管 VS1838B（学习遥控信号）

| VS1838B 引脚 | 连接到 | 说明 |
|-------------|--------|------|
| VCC（右脚） | 3.3V（Pin 17） | 模块供电 |
| GND（中脚） | GND（Pin 9） | 公共地 |
| OUT（左脚） | GPIO24（Pin 18） | 信号输出 |

> VS1838B 平面朝向自己时，从左到右依次是 OUT、GND、VCC。  
> 学习信号时将遥控器对准 VS1838B 平面，距离 5~10cm 效果最好。

### 6.4 温湿度传感器 + 光照传感器 + 光敏电阻

**示意图：**

![传感器接线](assets/wiring_sensors.png)

#### 🟣 DHT22 温湿度传感器

| DHT22 引脚 | 连接到 | 说明 |
|-----------|--------|------|
| VCC（1脚） | 3.3V（Pin 1） | 模块供电 |
| DATA（2脚） | GPIO4（Pin 7） | 数据信号 |
| NC（3脚） | 不接 | 空脚 |
| GND（4脚） | GND（Pin 9） | 公共地 |

> **必须在 VCC 和 DATA 之间接一个 4.7kΩ 上拉电阻**，否则读取失败。  
> 电阻一端接面包板 3.3V 轨，另一端接 DATA 所在行。

#### 🟤 BH1750 光照传感器（GY-302）

| BH1750 引脚 | 连接到 | 说明 |
|------------|--------|------|
| VCC | 3.3V（Pin 17） | 模块供电 |
| GND | GND（Pin 6） | 公共地 |
| SCL | GPIO3/SCL（Pin 5） | I2C 时钟 |
| SDA | GPIO2/SDA（Pin 3） | I2C 数据 |
| ADDR | GND | 地址选择（接 GND = 0x23，接 VCC = 0x5C） |

**验证 BH1750 是否正常：**
```bash
sudo i2cdetect -y 1
# 正常时在 23 位置显示 23（即地址 0x23）
```

#### ⚪ 光敏电阻模块（灯光状态反馈）

| 模块引脚 | 连接到 | 说明 |
|---------|--------|------|
| VCC | 3.3V（Pin 17） | 模块供电 |
| GND | GND（Pin 25） | 公共地 |
| DO | GPIO23（Pin 16） | 数字输出（亮=LOW，暗=HIGH） |

> 模块上的**蓝色电位器**用于调节灵敏度：  
> - 顺时针旋转 → 灵敏度提高（更容易判定为"亮"）  
> - 逆时针旋转 → 灵敏度降低  
> 调节方法：开灯后旋转电位器直到模块上的指示 LED 亮起，即为合适灵敏度。

**安装位置：** 固定在灯光照射范围内，避免被遮挡。用于确认舵机按压开关后灯是否真的亮起（闭环控制）。

```
  5. 温度-        [✗ 未录制]
  q. 退出

输入选项: 1
准备学习: 空调开机-制冷
请将遥控器对准红外接收管，然后按下对应按键...
```

每个信号学习完成后会显示"学习成功"，信号保存到 `ir_codes.json`。

> 至少需要学习 **空调开机-制冷**、**空调开机-制热**、**空调关机** 三个信号。

## 10. 设置开机自启

让系统开机后自动运行，不需要每次手动启动。

### 10.1 创建 systemd 服务

```bash
sudo nano /etc/systemd/system/smartroom.service
```

粘贴以下内容（注意修改用户名和路径）：

```ini
[Unit]
Description=Smart Room Control System
After=network.target

[Service]
Type=simple
User=pi
WorkingDirectory=/home/admin/smart-room-control
ExecStart=/home/admin/smart-room-control/venv/bin/python main.py
Restart=on-failure
RestartSec=5
StandardOutput=journal
StandardError=journal

[Install]
WantedBy=multi-user.target
```

保存退出（`Ctrl+O`，`Ctrl+X`）。

### 10.2 启用服务

```bash
# 重新加载 systemd 配置
sudo systemctl daemon-reload

# 启用开机自启
sudo systemctl enable smartroom.service

# 立即启动
sudo systemctl start smartroom.service

# 查看运行状态
sudo systemctl status smartroom.service
```

看到 `Active: active (running)` 表示成功。

### 10.3 常用服务管理命令

```bash
# 查看实时日志
sudo journalctl -u smartroom.service -f

# 停止服务
sudo systemctl stop smartroom.service

# 重启服务（修改代码后使用）
sudo systemctl restart smartroom.service

# 禁用开机自启
sudo systemctl disable smartroom.service
```

## 11. 常见问题

### Q1：摄像头无法打开

```bash
# 检查摄像头是否被识别
ls /dev/video*
# 应该看到 /dev/video0

# 检查权限
sudo usermod -a -G video pi
# 重新登录后生效
```

### Q2：GPIO 权限错误（RuntimeError: No access to /dev/mem）

```bash
sudo usermod -a -G gpio pi
# 重新登录后生效
# 或者直接用 sudo 运行（不推荐长期使用）
```

### Q3：DHT22 读取失败

- 检查 4.7kΩ 上拉电阻是否接好
- DHT22 对时序敏感，偶尔失败是正常的，代码已有重试逻辑
- 确认 `TEMP_SENSOR_PIN = 4` 与实际接线一致

### Q4：BH1750 找不到（i2cdetect 没有显示 0x23）

```bash
# 确认 I2C 已启用
sudo raspi-config  # Interface Options → I2C → Yes

# 检查接线：SDA→Pin3, SCL→Pin5
# ADDR 引脚接 GND 时地址为 0x23，接 VCC 时为 0x5C
```

### Q5：舵机抖动或不动

- 确认舵机接 5V 而不是 3.3V
- 在 `config.py` 中调整 `SERVO_ANGLE_ON` 和 `SERVO_ANGLE_OFF` 的角度值
- 先用小角度测试（如 80° 和 100°），找到刚好能按下开关的角度

### Q6：红外发射无效（空调没反应）

- 确认已完成红外信号学习（`ir_codes.json` 文件存在且不为空）
- 将红外发射管对准空调接收窗（通常在空调正面右下角）
- 距离保持在 3 米以内
- 检查三极管接线是否正确

### Q7：Web 界面无法访问

```bash
# 确认服务在运行
sudo systemctl status smartroom.service

# 确认端口在监听
ss -tlnp | grep 8080

# 检查防火墙（通常树莓派默认不开防火墙）
sudo ufw status
```

### Q8：YOLO 模型下载失败

```bash
# 手动下载（在有网络的环境）
python3 -c "from ultralytics import YOLO; YOLO('yolov8s.pt')"

# 如果网络不好，可以在 PC 上下载后用 SCP 传到树莓派
# 模型文件放在 smart-room-control/ 目录下即可
```

### Q9：内存不足（树莓派卡死）

树莓派 5 8GB 内存充裕，但如果同时运行多个大模型可能有压力：

```bash
# 查看内存使用
free -h

# 如果内存紧张，在 config.py 中换回轻量模型
YOLO_MODEL = "yolov8n.pt"
```
