# 硬件接线与自检指南（树莓派5）

这份文件用于解决两个问题：

1. 每个模块接到树莓派哪个物理针脚、哪个 BCM GPIO。
2. 接线后如何确认程序真的读到了传感器，而不是仍在模拟模式。

## 1. 先确认程序是否读取真实硬件

项目现在的 `config.py` 已改为自动判断：

- Windows / 普通 PC：默认 `SIMULATE_GPIO=True`，走模拟数据。
- 树莓派 Linux ARM：默认 `SIMULATE_GPIO=False`，读取真实硬件。
- 强制真实硬件：`export SMART_ROOM_SIMULATE_GPIO=0`
- 强制模拟：`export SMART_ROOM_SIMULATE_GPIO=1`

在树莓派里进入虚拟环境后执行：

```bash
cd ~/smart-room-control
source ~/smart-room-control/bin/activate
python -c "import config; print(config.SIMULATE_GPIO)"
```

输出必须是 `False`，否则 GPIO 不会生效。

树莓派5如果 `import RPi.GPIO` 报错，可先尝试安装兼容层：

```bash
pip install rpi-lgpio
python -c "import RPi.GPIO as GPIO; print('GPIO OK')"
```

## 2. 树莓派5 40Pin 总表（本项目用到的针脚）

注意：程序里使用的是 BCM 编号，不是物理 Pin 编号。接线时请同时核对两列。

| 模块 | 模块引脚 | 树莓派 BCM | 物理 Pin | 接法 |
| --- | --- | ---: | ---: | --- |
| 舵机 MG90S | 信号线（橙/黄） | GPIO18 | Pin 12 | 直接接树莓派信号脚 |
| 舵机 MG90S | VCC（红） | 5V | Pin 2 或 4 | 建议外部 5V 供电，必须共地 |
| 舵机 MG90S | GND（棕/黑） | GND | Pin 6/9/14/20/25/30/34/39 | 与树莓派共地 |
| 继电器 | IN | GPIO27 | Pin 13 | 直接接树莓派信号脚 |
| 继电器 | VCC | 5V | Pin 2 或 4 | 看继电器模块要求 |
| 继电器 | GND | GND | 任一 GND | 共地 |
| DHT22/AM2302 | DATA | GPIO4 | Pin 7 | DATA 与 3.3V 间加 4.7kΩ 上拉 |
| DHT22/AM2302 | VCC | 3.3V | Pin 1 或 17 | 推荐 3.3V |
| DHT22/AM2302 | GND | GND | 任一 GND | 共地 |
| BH1750/GY-302 | SDA | GPIO2/SDA | Pin 3 | I2C 数据 |
| BH1750/GY-302 | SCL | GPIO3/SCL | Pin 5 | I2C 时钟 |
| BH1750/GY-302 | VCC | 3.3V | Pin 1 或 17 | 推荐 3.3V |
| BH1750/GY-302 | GND | GND | 任一 GND | 共地 |
| 光敏电阻模块 | DO | GPIO23 | Pin 16 | 数字输出，程序只读 DO |
| 光敏电阻模块 | VCC | 3.3V | Pin 1 或 17 | 推荐 3.3V，避免 5V 输出伤 GPIO |
| 光敏电阻模块 | GND | GND | 任一 GND | 共地 |
| 红外接收 VS1838B | OUT | GPIO24 | Pin 18 | 注意不同模块引脚顺序可能不同 |
| 红外接收 VS1838B | VCC | 3.3V | Pin 1 或 17 | 推荐 3.3V |
| 红外接收 VS1838B | GND | GND | 任一 GND | 共地 |
| 红外发射管 | 驱动输入 | GPIO25 | Pin 22 | 建议 GPIO25 经 1kΩ 到三极管基极 |

## 3. 面包板还是直接连接

推荐原则：

- 传感器类（DHT22、BH1750、光敏模块、红外接收）：插面包板，便于统一接 3.3V 和 GND。
- 执行器类（舵机、继电器、红外发射）：可以接面包板，但电源线更建议独立、牢靠，特别是舵机不要长期直接吃树莓派 5V。
- 所有外部电源的 GND 必须和树莓派 GND 连在一起，否则信号脚没有共同参考电平。

面包板电源轨建议：

| 电源轨 | 电压 | 接哪些模块 |
| --- | --- | --- |
| 红色 + | 3.3V | DHT22、BH1750、光敏模块、红外接收 |
| 蓝色 - | GND | 所有传感器 GND、树莓派 GND、外部 5V GND |
| 单独 5V | 5V | 舵机、继电器、红外发射驱动电路 |

不要把树莓派 GPIO 输入脚直接接 5V。GPIO 输入最高按 3.3V 处理。

## 4. 每个部件的单独验证方法

先运行自检脚本：

```bash
python hardware_check.py
python hardware_check.py --watch
```

`--watch` 模式下，每 2 秒刷新一次。你可以遮住光敏电阻、照射光敏电阻、按空调遥控器，看读数是否变化。

### 光敏电阻模块

接法：

- VCC -> 3.3V
- GND -> GND
- DO -> GPIO23 / Pin16

验证：

```bash
python hardware_check.py --watch
```

正常现象：

- 光照变化时，`光敏反馈 DO: BCM23 = HIGH/LOW` 会切换。
- 多数模块是“亮=LOW，暗=HIGH”。如果你的模块相反，改 `config.py`：

```python
LIGHT_FEEDBACK_ACTIVE_LOW = False
```

### BH1750 光照传感器

接法：

- VCC -> 3.3V
- GND -> GND
- SDA -> GPIO2 / Pin3
- SCL -> GPIO3 / Pin5

验证：

```bash
sudo raspi-config
# Interface Options -> I2C -> Enable
i2cdetect -y 1
python hardware_check.py
```

正常现象：

- `i2cdetect -y 1` 能看到 `23` 或 `5c`。
- 自检脚本显示 `BH1750: I2C地址=0x23 光照=... lux`。

### DHT22 温湿度传感器

接法：

- VCC -> 3.3V
- DATA -> GPIO4 / Pin7
- GND -> GND
- DATA 和 3.3V 之间加 4.7kΩ 上拉电阻。

验证：

```bash
python hardware_check.py
```

DHT22 偶尔读取失败是正常的；连续失败通常是 DATA 接错、没有上拉、电源不稳或模块引脚顺序看反。

### 红外接收管 VS1838B

接法：

- OUT -> GPIO24 / Pin18
- VCC -> 3.3V
- GND -> GND

验证：

```bash
python hardware_check.py --watch
python ir_learn_tool.py
```

注意：VS1838B 裸管和模块板的引脚顺序不一定一样，必须按你手上器件的丝印或资料核对。

### 红外发射管

推荐不要让 GPIO 直接硬推红外发射管，距离会很短，也可能电流不合适。

推荐接法：

- GPIO25 / Pin22 -> 1kΩ 电阻 -> S8050/2N2222 基极
- 三极管发射极 -> GND
- 红外发射管正极 -> 5V 串 100Ω 电阻
- 红外发射管负极 -> 三极管集电极
- 外部 5V GND -> 树莓派 GND

验证：

```bash
python ir_learn_tool.py
python main.py
```

先学习遥控器信号，再让程序发射。

## 5. 程序和网页端是否会自动读取零件

会，但前提是：

1. `config.SIMULATE_GPIO` 为 `False`。
2. 对应依赖安装成功。
3. 传感器接在 `config.py` 配置的 BCM 引脚上。
4. 启动的是 `main.py` 或 `gui_app.py`，这两个入口都会创建并启动 `TemperatureSensor`、`LightSensor`、`DeviceManager`。

读取链路如下：

| 零件 | 代码模块 | 页面/GUI显示 |
| --- | --- | --- |
| DHT22 | `sensor.py` | 温度、湿度 |
| BH1750 | `light_sensor.py` 的 `LightSensor` | 光照 lux |
| 光敏电阻 DO | `light_sensor.py` 的 `LightFeedbackSensor` | 灯光实际反馈 |
| 舵机 | `servo_switch.py` | 手动/自动开关灯时动作 |
| 继电器 | `relay_controller.py` | 电源开关 |
| 红外接收/发射 | `ir_controller.py` | 红外学习、空调控制 |

网页端本身不直接读 GPIO；它通过 Flask API 读取后端状态。只要后端真实读到传感器，网页会自动刷新显示。

## 6. 最常见错误排查

| 现象 | 优先检查 |
| --- | --- |
| 所有传感器都有数值但怎么遮挡都不变 | `SIMULATE_GPIO` 仍是 `True` |
| 光敏模块板上灯亮，但程序不变 | DO 接错、接到了 AO、VCC 用了 5V、`LIGHT_FEEDBACK_ACTIVE_LOW` 方向相反 |
| BH1750 没读数 | I2C 未启用、SDA/SCL 对调、地址是 0x5C、VCC/GND 接反 |
| DHT22 一直失败 | 没有 4.7kΩ 上拉、DATA 不是 GPIO4、供电不稳 |
| 红外学习超时 | 接收管 OUT/GND/VCC 顺序看反，或遥控器没对准 |
| 舵机不动或树莓派重启 | 舵机电流过大，改用外部 5V 供电并共地 |
