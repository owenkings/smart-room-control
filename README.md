# 智能教室/实验室 网络化测控系统

课程《网络化测控技术》项目 — 基于 YOLO 人员检测的非侵入式智能教室网络化测控原型系统。

## 一、项目定位

**本项目设计并实现了一套面向普通教室的非侵入式智能测控原型系统。**
系统利用 YOLO 进行人员检测，结合温湿度与光照环境信息，通过舵机、红外发射和继电器等执行模块，实现对教室灯光与空调的自动化控制，并通过 Web 页面完成状态监控、手动接管和日志展示。

## 二、系统架构

```
┌─ 感知层 ─────────────────────┐
│ USB摄像头 → YOLOv8人员检测   │
│ DHT22    → 温度湿度          │
│ BH1750   → 环境光照(决策用)  │
│ 光敏电阻 → 灯光实际状态反馈  │
└──────┬────────────────────────┘
       ↓
┌─ 决策层 ─────────────────────┐
│ PresenceFilter 检测防抖      │
│ DecisionEngine 多源融合判断  │
│ AUTO/MANUAL 双模式切换       │
└──────┬────────────────────────┘
       ↓
┌─ 执行层 ─────────────────────┐
│ MG90S舵机 → 按压墙壁灯开关  │
│ 红外发射  → 空调遥控信号    │
│ 继电器    → 低压设备电源    │
└──────┬────────────────────────┘
       ↓
┌─ 网络展示层 ─────────────────┐
│ Flask B/S监控界面            │
│ 实时视频 + 状态 + 日志 + 控制│
└─ 数据持久化 ─────────────────┘
  logs/system.log (运行日志)
  logs/control_history.csv (CSV实验数据)
```

## 四、快速开始

### 阶段1: PC 端验证(无需硬件)

```bash
cd smart-room-control
pip install -r requirements.txt

# 先跑验证脚本
python test_run.py

# 所有测试通过后启动主程序
python main.py
# 浏览器访问 http://localhost:8080
```

### 阶段2: 树莓派部署

```bash
# 在树莓派上
pip install -r requirements.txt
pip install RPi.GPIO adafruit-circuitpython-dht smbus2

# 树莓派会自动使用真实GPIO；确认输出为 False
python -c "import config; print(config.SIMULATE_GPIO)"

# 如果需要手动强制真实硬件模式
export SMART_ROOM_SIMULATE_GPIO=0

# 启用I2C(用于BH1750)
sudo raspi-config  # Interface Options → I2C → Enable

# 逐项检查接线
python hardware_check.py
python hardware_check.py --watch

# 运行
python main.py
```

更详细的接线、面包板供电和逐项自检说明见 `硬件接线与自检指南_树莓派5.md`。

## 六、验证项目是否跑通

### 验证 1: 模块加载
运行 `python test_run.py`，看到全部 `[PASS]` 表示代码层无问题。

### 验证 2: Web 服务启动
运行 `python main.py`，日志出现:
```

```
浏览器打开后能看到界面、人数显示、温湿度数据在变化。

### 验证 3: 决策链路
- PC 模拟模式下，温湿度会随机波动，当温度 >28°C 会自动触发空调
- 通过 Web 手动控制灯光/电源，日志区会看到"[手动]"记录
- 切换"手动模式"后，自动决策暂停

### 验证 4: 硬件接入后(树莓派)
- 站到摄像头前 → 舵机转动按压开关(有物理动作) → 光敏电阻反馈"灯亮"
- 离开超过 30 秒 → 舵机反向按压 → 灯灭
- 学习红外信号后 → 空调能被控制

### 验证 5: 实验数据
运行一段时间后查看 `logs/control_history.csv`，可用 Excel 打开分析。

## 七、核心功能特性

- **检测防抖**: 滞回滤波器避免单帧误触发(连续2次检测到才开灯，连续5次未检测到才关)
- **节能策略**: 有人 + 光照<150lux 才开灯，避免白天无谓浪费
- **闭环控制**: 舵机按压后读光敏电阻确认实际状态，不符则重试
- **双模式**: AUTO(自动决策) / MANUAL(仅手动控制) 一键切换
- **多源融合**: YOLO + 温度 + 光照 + 光敏反馈 联合决策
- **数据留痕**: system.log + control_history.csv 双重日志
- **红外学习**: Web 界面直接学习空调遥控器信号

## 八、答辩演示建议

### 演示顺序
1. 打开 Web 页面，介绍系统架构
2. 摄像头无人 → 系统显示"无人"
3. 人进入画面 → YOLO检测到人 → 滤波器计数 → 稳定后判定"有人"
4. 光照不足时自动开灯(舵机动作)，光敏电阻反馈"灯亮"
5. 温度超过阈值 → 红外发射模拟遥控器开空调
6. 人离开 → 无人超时 → 自动关闯、断电、关空调
7. 切换到手动模式 → 自动决策暂停，演示手动接管
8. 展示 CSV 实验数据

### 话术要点
- **定位**: 非侵入式改造原型系统(不接入教室强电)
- **创新点**: 舵机+光敏反馈形成闭环 / 多源传感融合决策 / AUTO/MANUAL 双模
- **局限说明**: 红外控制针对固定型号空调学习适配 / 继电器仅用于低压设备演示
