## 一、BH1750 光照传感器 — 未实现

### 现象
- `i2cdetect -y 1` 全 `--`，总线上无设备响应
- `hw_check.py bh1750` 扫描 i2c-1/13/14 均未发现 0x23 或 0x5C

### 根本原因（待确认）
I2C 总线通了但 BH1750 无响应，最可能是：
1. SDA（Pin 3）或 SCL（Pin 5）杜邦线接触不良 / 插错行
2. VCC 未真正供到 3.3V（面包板电源轨跳线未拨到 3.3V）
3. 模块本身损坏（概率低）

### 解决方向
- [ ] 用万用表测 BH1750 模块 VCC 引脚对 GND 是否有 3.3V
- [ ] 拔掉 SDA/SCL 杜邦线重新插，确认插入面包板的行与树莓派 Pin 3/5 对应
- [ ] 换一根杜邦线测试（排除断线）
- [ ] 确认 `sudo raspi-config → Interface → I2C → Yes` 已启用并重启
- [ ] 代码层面无需修改，`light_sensor.py` 实现正确

## 二、DHT22 温湿度传感器 — 未实现

### 现象
- 初始化通过，但连续 10 次读取均失败
- `GPIO4 = 0` 持续为低电平（正常悬空应为高电平）

### 根本原因（已基本确认）
`GPIO4 持续为 0` 说明 DATA 线被强制拉低，最可能是：
1. **DATA 和 GND 接反**（最常见）
2. DATA 线短路到 GND（插在同一横排）
3. 模块没有内置上拉电阻，需要外接 10kΩ

### 解决方向
- [ ] 重新确认 DHT22 模块引脚顺序（看丝印，不同厂家顺序不同）
- [ ] 确认三个脚在面包板的**不同横排**
- [ ] 在 VCC 行和 DATA 行之间加 10kΩ 上拉电阻
- [ ] 确认面包板 GND 轨已连接树莓派 GND（共地）
- [ ] 代码层面无需修改，`sensor.py` 实现正确

## 三、红外接收/发射 — 未实现

### 现状
- VS1838B 接收管已接线，未做功能验证
- 红外发射管当前为直接接法（GPIO25 → 220Ω → 发射管），无三极管驱动
- `ir_controller.py` 代码已实现学习+重放逻辑

### 已知限制
- 直接接法发射距离约 1~2 米，控制空调可能不够
- Python 用 `time.sleep()` 模拟 38kHz 载波，精度受系统调度影响，可能导致部分空调不识别

### 解决方向
- [ ] **第一步**：先验证 VS1838B 能接收信号
  ```bash
  python hw_check.py ir_recv
  # 5秒内按遥控器任意键，看是否检测到电平变化
  ```
- [ ] **第二步**：验证发射管能被接收管检测到（近距离）
  ```bash
  python hw_check.py ir_send
  ```
- [ ] **第三步**：运行红外学习工具录制空调信号
  ```bash
  python ir_learn_tool.py
  ```
- [ ] **后续优化**：加 S8050 NPN 三极管驱动电路，提升发射距离到 5 米
  ```
  GPIO25 → 1kΩ → 三极管基极(B)
  5V → 红外发射管正极(长脚)
  红外发射管负极(短脚) → 三极管集电极(C)
  三极管发射极(E) → GND
  ```
- [ ] **备选方案**：使用 `pigpio` 库替代 `time.sleep()` 实现精确 38kHz 载波
  ```bash
  sudo apt install pigpio
  pip install pigpio
  # 启动 pigpio 守护进程
  sudo pigpiod
  ```

## 四、舵机控制逻辑重设计

### 当前问题
现有代码直接用固定角度（60°开/120°关）控制，没有用户校准流程，
实际安装时开关位置各不相同，固定角度不适用。

### 设计目标
用户通过界面完成一次性校准，之后自动执行：

```
校准流程（一次性）:
  1. 用户手动将舵机臂调到"中立位置"（不碰开关）
  2. 点击"设为复位位置" → 记录当前角度为 NEUTRAL（如 90°）
  3. 点击"正转"微调 → 舵机臂碰到开关并按下 → 点击"设为开灯位置"
  4. 点击"反转"微调 → 舵机臂碰到开关另一侧并按下 → 点击"设为关灯位置"
  5. 保存到 servo_calibration.json

执行流程（每次触发）:
  开灯: 复位位置 → 开灯位置（按下）→ 复位位置（松开）
  关灯: 复位位置 → 关灯位置（按下）→ 复位位置（松开）
```

### 角度范围说明
```
舵机物理范围: 0° ~ 180°
双翼舵机臂实际有效范围: 约 ±45°（即 45° ~ 135°）
建议复位位置: 90°（中立）
建议开灯范围: 90° → 60°（向一侧按压）
建议关灯范围: 90° → 120°（向另一侧按压）
具体角度需根据实际安装位置校准
```

### 需要修改的文件
- [ ] `servo_switch.py`：增加 `calibrate()` 方法，支持动态设置三个角度
- [ ] `config.py`：`SERVO_ANGLE_ON/OFF/NEUTRAL` 改为从 `servo_calibration.json` 读取
- [ ] `web_server.py`：增加 `/api/servo/calibrate` 接口
- [ ] `gui_app.py`：增加舵机校准面板（微调按钮 + 设置位置按钮）
- [ ] `templates/index.html`：增加 Web 端舵机校准界面

### servo_calibration.json 格式
```json
{
  "neutral": 90,
  "on": 60,
  "off": 120,
  "action_duration": 0.5,
  "calibrated": true,
  "calibrated_at": "2026-05-01 12:00:00"
}
```
